# Security Policy

## Supported Versions

We provide security updates and fixes for the following versions of this project. If you're using a version that is not listed, please consider upgrading to a supported version.

| Version | Supported          |
| ------- | ------------------ |
| 5.1.x   | :white_check_mark: |
| 5.0.x   | :x:                |
| 4.0.x   | :white_check_mark: |
| < 4.0   | :x:                |

## Reporting a Vulnerability

We take security seriously and encourage you to report any vulnerabilities you discover. To report a vulnerability, please follow these steps:

1. **Submit a Report**: Send an email to [contact@johnlloydserrano.com](mailto:contact@johnlloydserrano.com) with detailed information about the issue.
2. **Provide Details**: When reporting, please include:
   - A clear description of the vulnerability.
   - Steps to reproduce (if possible).
   - Any relevant logs or screenshots.
3. **Response Time**: We aim to respond to all vulnerability reports within 48 hours, acknowledging receipt and providing an initial assessment.
4. **Resolution and Updates**: If your vulnerability report is accepted, we will work to provide a fix as quickly as possible, and we will notify you when the fix is released. The vulnerability will be patched in the next available release of the project.
5. **Declined Reports**: If your vulnerability report is declined, we will provide an explanation of why it was not accepted. If the report is deemed a false positive or not applicable to the project, we will clarify that as well
